# Phase 0 ACT-015A CEM / Elite-Sampling Optimizer Report

## Summary

- Goal: test whether the successful ACT-013 cold top-k handoff can be moved inside the optimizer, instead of only being used as a terminal decoding/reranking step.
- Method: add two CEM variants in the reduced ProtenixMini setting: `CEMp`, which ranks elite hard samples by BT PAE, and `CEMc`, which ranks elite hard samples by contact loss.
- Main answer: not yet. This simple CEM implementation makes the distribution more discrete, but it does not improve hard candidate quality. Under matched 24-candidate budget, `CEMp` reaches BT PAE 14.9997 while M3 reaches 10.7001 and M7c reaches 10.8029.
- Interpretation: elite sampling from the initial relaxed distribution is too weak as a proposal mechanism. The optimizer collapses around low-quality hard candidates rather than discovering the soft high-quality basin that M3/M7c can reach through gradient updates.
- Decision: do not scale this CEM variant. The next useful experiment should either warm-start CEM from M3/M7c terminal distributions or move to ACT015B QP-style constrained gradient updates.

## 1. Experiment Motivation

The project hypothesis is that Mosaic already has multiple oracle losses, but its use of them is too coarse. We want more precise geometry-aware optimization that improves discrete binder candidates, not just relaxed soft distributions.

ACT013 showed that cold top-k handoff can rescue some relaxed gains: sampling several hard candidates from a terminal relaxed distribution and reranking them can outperform direct argmax. That suggested a natural question:

> If terminal cold top-k helps, can we internalize the same idea as an optimizer by repeatedly sampling hard candidates, selecting elites, and updating the relaxed distribution toward them?

This is the ACT015A question. It directly tests the user's suggested CEM direction.

## 3. Core Algorithm or Method

`CEMp` and `CEMc` start from a relaxed amino-acid distribution. At each round:

1. sample 8 hard sequences using cold top-k sampling;
2. score every hard sequence with the same ProtenixMini candidate scorer used for M3/M7c handoff;
3. select the top 2 elites by a ranking metric;
4. update the relaxed distribution toward the empirical elite distribution, with 0.05 uniform mixing;
5. repeat for 3 rounds, then score all samples plus the final argmax.

The only difference between the two CEM variants is the elite metric:

- `CEMp`: minimize BT PAE.
- `CEMc`: minimize contact loss.

This deliberately uses hard-candidate oracle calls inside the optimizer. Therefore the fair comparison is not soft M3/M7c alone, but M3/M7c cold top-k under the same candidate scoring budget.

## 6. How to Read the Tables

The candidate mean table describes the average quality of all scored candidates under each method and score mode. It is useful for distribution-level behavior but is not the final selection criterion.

The best-of-budget tables are the more important test. They mimic a practical screening pipeline: from the first `K` hard samples per seed, choose the candidate that optimizes a reranking metric. A positive CEM result would require CEM to beat M3/M7c at the same `K`, especially at `K=24`, where all CEM samples are available.

The round table diagnoses mechanism. Entropy reduction confirms that CEM is making the relaxed distribution discrete; the BT PAE/contact columns reveal whether that commitment is useful.

## 7. Interpretation

The experiment answers the immediate ACT015A question:

> Does a simple CEM / elite-sampling optimizer turn cold top-k handoff into a better optimizer?

Current answer: no.

The strongest evidence is the matched best-of-24 comparison. With the same hard-candidate budget, M3 obtains BT PAE 10.7001 and M7c obtains 10.8029, while CEMp obtains 14.9997 and CEMc obtains 15.8457. BT ipTM shows the same pattern: M3 0.4671, M7c 0.4158, CEMp 0.2244, CEMc 0.2111.

The likely reason is proposal quality. CEM samples hard candidates directly from an initially broad distribution, then reinforces the best among a poor local sample set. M3/M7c first use relaxed gradient optimization to move the distribution into a better basin, and only then use cold top-k sampling. ACT015A removed that gradient-guided basin-finding step, so CEM committed too early.

This result is still useful. It says the positive ACT013 result should not be interpreted as "sampling alone solves the discrete gap." The useful part of ACT013 was probably the combination of relaxed gradient descent plus cold candidate selection, not hard-sample elite selection by itself.

## 8. Conclusion and Discussion

### What was the goal?

The goal was to test whether CEM could internalize cold top-k handoff and directly optimize expected hard-candidate quality.

### Did we get the answer?

Yes for this minimal variant: CEM is not competitive. It makes sequences more discrete, but its selected hard candidates are worse than M3/M7c under matched candidate budget.

### How did we find it?

We implemented `scripts/run_protenix_cem_optimizer.py`, ran it on Quest H100 with two seeds, compared CEMp/CEMc against M3/M7c using the same ProtenixMini candidate scorer, and analyzed best-of-1/8/24 selection under BT PAE, BT ipTM, and contact reranking.

### Are the results correct?

The run completed successfully with exit status 0. The comparisons use the same target, binder length, candidate sampling temperature, top-k sampling size, seeds, Protenix cache, and reranking script. The result is internally consistent across BT PAE, BT ipTM, and contact reranking.

The result should still be treated as a Phase 0 diagnostic, not a final negative result for all CEM-like methods. It uses only two seeds, a short 24-residue binder, ProtenixMini, and a simple CEM update. It rules out this naive CEM design as the next thing to scale; it does not rule out warm-started or gradient-hybrid CEM.

## 9. Limitations and Caveats

- Small run: only 2 seeds and a reduced ProtenixMini setting.
- CEM was initialized from a random relaxed distribution, not from an M3/M7c terminal distribution.
- CEMp and CEMc share the same first-round samples per seed, so budget-1 and budget-8 rows are identical for the two CEM methods; this is expected because method-specific CEM updates begin after round 0.
- The CEM update uses elite empirical frequency with uniform mixing. It has no trust region against reinforcing bad early samples beyond the fixed update rate and uniform floor.
- This run does not test QP-style constrained updates or gradual position-wise hardening.

## 10. Next Steps

1. Do not scale naive ACT015A CEM.
2. Try a warm-started CEM diagnostic only if needed: run M3 or M7c for 3 steps, then apply 1-2 CEM rounds from that terminal distribution.
3. Prioritize ACT015B: QP-style constrained update that preserves contact descent while limiting auxiliary harm.
4. Keep ACT015C position-wise hardening as the direct response to terminal argmax collapse.
